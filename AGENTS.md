## Version Control
- never make changes to main directly
  - pull main and create a branch before making changes
  - create a pull request when changes are finalized
- create small commits of complete units of work
- use conventional commit messages (but not for pull reequest names), and keep them concise

## Python
- keep things simple
- when running tests, use pipenv run pytest
- when installing dependencies, use pipenv
- before committing, ensure no type checking errors, run tests and pre-commit hooks

## Repo Specific
- provide reliable sources (internet links) for tax calculations for any new functions created
